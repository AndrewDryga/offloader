# Plain-Language Public Copy

Public copy should explain the buyer's situation before it introduces product
terms. Lead with concrete nouns: an API request, a warehouse query, a snapshot,
a REST endpoint, a bill. Then name the abstraction.

Avoid dense strings like "warehouse-built product data", "bounded production
read APIs", or "governed serving layer" in first-touch copy unless the sentence
also explains them in normal language. Prefer shorter sentences and one new idea
per sentence.

When a term is useful, define it through the user's workflow: the warehouse builds
the data once, Offloader serves it many times until the next snapshot is ready.
